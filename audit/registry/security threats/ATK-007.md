# ATK-007: Free-list and cache lifetime bugs permit use-after-free and double free

**CASSE:** Data • Poisoning • Core

**HDF5 SSP vulnerability tags:** `LIB|FMT`

## Preconditions

- Crafted file state or parser error handling reaches free-list garbage collection, cache unpin, or reallocation paths.
- The process continues operating on metadata structures after an earlier failure or teardown transition.

## Trigger / entry point

Malformed inputs or edge-case control flow cause memory to be freed, reused, or reallocated inconsistently inside shared allocation and cache-management code.

## Vulnerability mechanism (CWE-style)

The library uses pointers after free, frees the same allocation twice, or mismanages object lifetime across cache and free-list transitions. Relevant CWEs include `CWE-416` and `CWE-415`.

## Expected impact

Heap corruption, hard-to-reproduce crashes, and credible code-execution primitives in long-lived services that parse many files in one process.

## Risk assessment

Risk scoring used: **Severity (1-5) x Likelihood (1-5) = Risk (1-25)**

Risk bands: **1-5 Low, 6-10 Moderate, 11-15 High, 16-20 Very High, 21-25 Critical**

### Severity

**5 - Critical.** Use-after-free and double-free bugs are strong exploitation primitives when they become controllable.

### Likelihood

**3 - Possible.** These issues usually need more specific control flow than simple malformed-field overflows, but they have recurred in allocator and cache code that many parse paths share.

**Risk:** **15 (High)**

## Exploitability notes

These bugs often look "local" in advisories, yet they matter for any service that opens attacker-supplied files repeatedly because lifetime corruption becomes more exploitable in persistent processes.

## Detection

ASan with quarantine enabled, targeted fault-injection around allocation failure and cache eviction, and regression tests that keep operating after parser errors instead of exiting immediately.

## Mitigations

Adopt stricter ownership rules for cache entries and free-list nodes, null out or invalidate freed pointers aggressively, and simplify allocator teardown paths that currently rely on subtle ordering assumptions.

## Tests / evidence

- Regression tests for `H5FL__blk_gc_list`, `H5FL__reg_gc_list`, `H5MM_realloc`, and `H5AC_unpin_entry`.
- Fault-injection tests that force partial failure, cache eviction, and repeated open/close cycles.
- CVE evidence from `CVE-2025-2913`, `CVE-2025-6856`, `CVE-2025-2925`, and `CVE-2021-46242`.
