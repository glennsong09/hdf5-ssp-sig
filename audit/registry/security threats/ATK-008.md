# ATK-008: Stack-based overflows in decode and traversal logic crash the parser

**CASSE:** Data • Poisoning • Core

**HDF5 SSP vulnerability tags:** `FMT|LIB`

## Preconditions

- The attacker can supply metadata or selections that drive deep traversal, stack-based formatting, or fixed-size stack buffers.
- The application exercises reference decoding, group traversal, selection mapping, or error-reporting paths.

## Trigger / entry point

Malformed metadata or extreme dimensions cause stack-based copies or recursion-related paths to exceed safe bounds.

## Vulnerability mechanism (CWE-style)

The library copies attacker-influenced data into fixed-size stack storage or consumes unbounded stack through recursive or nested decode logic. Relevant CWEs include `CWE-121` and `CWE-674`.

## Expected impact

Denial of service, instruction-pointer corruption, and context-dependent code-execution potential when stack protections are weak or bypassed.

## Risk assessment

Risk scoring used: **Severity (1-5) x Likelihood (1-5) = Risk (1-25)**

Risk bands: **1-5 Low, 6-10 Moderate, 11-15 High, 16-20 Very High, 21-25 Critical**

### Severity

**5 - Critical.** Stack corruption can terminate the process immediately and, in weaker hardening environments, can still become an execution primitive.

### Likelihood

**3 - Possible.** These paths are somewhat narrower than the most common heap-overflow cases, but the CVE history shows several independent reachable stack bugs.

**Risk:** **15 (High)**

## Exploitability notes

Modern stack protections reduce some exploitability, but they do not remove the availability risk and they are not guaranteed across every HPC, embedded, or legacy build environment.

## Detection

Compiler hardening with stack canaries, deep-structure fuzzing, recursion-depth tests, and negative tests that drive maximum selection and reference complexity under sanitizer builds.

## Mitigations

Replace fixed-size stack buffers with checked dynamic storage where practical, enforce depth and size limits for recursive structures, and keep stack-consuming diagnostics out of attacker-controlled hot paths.

## Tests / evidence

- Regression tests for `H5G__node_cmp3`, `H5R__decode_heap`, `H5FL_arr_malloc`, `H5E_printf_stack`, and `H5D__create_chunk_file_map_hyper`.
- Fuzz cases for oversized names, malformed references, and extreme selections.
- CVE evidence from `CVE-2025-6857`, `CVE-2024-29164`, `CVE-2024-29158`, `CVE-2024-32609`, and `CVE-2021-45833`.
