# ATK-003: Heap and shared-message deserialization corrupts core metadata state

**CASSE:** Data • Poisoning • Core

**HDF5 SSP vulnerability tags:** `FMT|LIB`

## Preconditions

- The file contains local heap, global heap, or shared-message structures controlled by an untrusted producer.
- The library follows heap pointers or reconstructs heap-backed metadata during open or lookup.

## Trigger / entry point

Heap records, free-list entries, or shared-message metadata are deserialized or deleted from crafted on-disk structures.

## Vulnerability mechanism (CWE-style)

The parser trusts heap element sizes, offsets, or record layout too early, causing heap-based overflows while rebuilding internal structures. Relevant CWEs include `CWE-122`, `CWE-787`, and `CWE-502` in the broad sense of unsafe deserialization of untrusted metadata.

## Expected impact

Crash, memory corruption, metadata poisoning across later reads, and possible code execution inside the parsing process.

## Risk assessment

Risk scoring used: **Severity (1-5) x Likelihood (1-5) = Risk (1-25)**

Risk bands: **1-5 Low, 6-10 Moderate, 11-15 High, 16-20 Very High, 21-25 Critical**

### Severity

**5 - Critical.** Heap-backed metadata is reused widely, so corruption here can compromise both integrity and execution safety.

### Likelihood

**4 - Likely.** The CVE history shows repeated issues across heap readers and heap cache reconstruction, which indicates a recurring trust-boundary weakness.

**Risk:** **20 (Very High)**

## Exploitability notes

Shared-message and heap structures are attractive targets because they are compact, pointer-rich, and often parsed before an application has validated any higher-level semantics of the file.

## Detection

Structure-aware fuzzing for heap objects, red-zone instrumentation around heap cache rebuild code, and corpus tests that mix valid object graphs with malformed heap offsets and payload lengths.

## Mitigations

Validate heap bounds before any copy, verify that referenced sub-objects remain inside the owning heap object, and maintain strict invariants for shared-message deletion and heap free-list reconstruction.

## Tests / evidence

- Regression tests for `H5HL__fl_deserialize`, `H5HG_read`, `H5HG__cache_heap_deserialize`, and `H5SM_delete`.
- Malformed-file tests with truncated heaps, overlapping heap items, and invalid free-list chains.
- CVE evidence from `CVE-2025-2924`, `CVE-2024-32612`, `CVE-2024-32613`, `CVE-2024-32621`, `CVE-2024-29157`, `CVE-2024-29160`, and `CVE-2025-2153`.
