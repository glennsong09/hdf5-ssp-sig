# ATK-006: Core vector-copy helpers amplify corruption across many data paths

**CASSE:** Data • Poisoning • Core

**HDF5 SSP vulnerability tags:** `LIB|FMT`

## Preconditions

- An attacker can supply metadata or dataset layouts that drive scatter/gather copies or vectorized memory movement.
- The application exercises reads, writes, or conversions that reach shared helper routines.

## Trigger / entry point

Shared memory-copy helpers calculate span lengths or destination extents incorrectly and then overrun buffers.

## Vulnerability mechanism (CWE-style)

Central utility functions perform copy or fill operations without proving that source and destination vector extents remain in bounds. Relevant CWEs include `CWE-122`, `CWE-787`, `CWE-125`, and `CWE-120`.

## Expected impact

Memory corruption or crash across otherwise unrelated features because the vulnerable primitive is reused broadly throughout the library.

## Risk assessment

Risk scoring used: **Severity (1-5) x Likelihood (1-5) = Risk (1-25)**

Risk bands: **1-5 Low, 6-10 Moderate, 11-15 High, 16-20 Very High, 21-25 Critical**

### Severity

**5 - Critical.** A bug in a shared copy primitive has wide blast radius and can be repurposed by many higher-level file constructs.

### Likelihood

**4 - Likely.** The attacker must reach the helper through a compatible dataset or metadata path, but multiple CVEs show that this is not a one-off call-site problem.

**Risk:** **20 (Very High)**

## Exploitability notes

This threat family is especially important because a single latent bug in a utility routine can survive targeted fixes elsewhere and reappear under new callers, making recurrence more likely than isolated parser defects.

## Detection

Unit tests around vector-bound calculations, fuzzing that mutates source and destination span tables, and sanitizer-backed tests that exercise shared helpers through multiple public API surfaces.

## Mitigations

Move span validation into one checked abstraction, assert total copied length against both source and destination capacity, and add cross-feature regression tests whenever a shared copy helper changes.

## Tests / evidence

- Regression coverage for `H5VM_memcpyvv`, `H5VM_array_fill`, and `H5D__scatter_mem`.
- Fuzz inputs that create fragmented selections, overlapping regions, and inconsistent vector lengths.
- CVE evidence from `CVE-2025-44904`, `CVE-2024-32614`, `CVE-2024-32605`, `CVE-2024-32623`, and `CVE-2024-33873`.
