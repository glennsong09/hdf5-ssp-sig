# ATK-004: Built-in filter pipelines overflow during decode and integrity checks

**CASSE:** Data • Poisoning • Core

**HDF5 SSP vulnerability tags:** `FMT|LIB`

## Preconditions

- An untrusted HDF5 file references a built-in filter such as Scale-Offset, N-Bit, or Fletcher32.
- The reader enables normal dataset decode behavior and reaches the filter pipeline.

## Trigger / entry point

Compressed or transformed chunk data is decoded, copied, or checksum-processed by built-in filter code.

## Vulnerability mechanism (CWE-style)

Filter code miscomputes output sizes, bit offsets, or copy ranges and then writes past destination buffers during decompression or verification. Relevant CWEs include `CWE-122`, `CWE-787`, and `CWE-120`.

## Expected impact

Crash, memory corruption, possible code execution, and loss of trust in dataset integrity checks because the validation path itself becomes attackable.

## Risk assessment

Risk scoring used: **Severity (1-5) x Likelihood (1-5) = Risk (1-25)**

Risk bands: **1-5 Low, 6-10 Moderate, 11-15 High, 16-20 Very High, 21-25 Critical**

### Severity

**5 - Critical.** Filter decoders operate on attacker-controlled bytes and sit directly in data ingest paths.

### Likelihood

**4 - Likely.** The attacker must select a vulnerable filter path, but the affected built-in filters are common enough that normal reads can trigger the issue without special deployment choices.

**Risk:** **20 (Very High)**

## Exploitability notes

Unlike external-plugin attacks, this class does not depend on plugin discovery or administrator error. A malicious file can reach the vulnerable logic using only built-in filter IDs and crafted chunk payloads.

## Detection

Coverage-guided fuzzing for filtered chunk decode, differential tests that compare decoded sizes against computed bounds, and sanitizer builds on every built-in filter path.

## Mitigations

Introduce strict per-filter size proofs before decode, reject impossible bit-width or chunk-size combinations, and treat checksum and decompression routines as parser attack surfaces that need the same hardening as metadata decoders.

## Tests / evidence

- Regression tests for `H5Z__filter_scaleoffset`, `H5Z__scaleoffset_decompress_one_byte`, `H5Z__nbit_decompress_one_byte`, and `H5Z__filter_fletcher32`.
- Fuzz seeds for malformed filtered chunks with undersized output buffers and invalid bit packing.
- CVE evidence from `CVE-2025-44905`, `CVE-2025-2308`, `CVE-2024-29159`, `CVE-2024-32615`, and `CVE-2024-29165`.
