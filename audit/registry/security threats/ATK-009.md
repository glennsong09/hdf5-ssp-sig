# ATK-009: Malformed files can exhaust memory or CPU and deny service

**CASSE:** Data • Poisoning • Core

**HDF5 SSP vulnerability tags:** `FMT|LIB|OPS`

## Preconditions

- The application accepts untrusted HDF5 input and parses it in a shared process or service.
- Resource limits are absent or are too loose for attacker-controlled allocation, cache, or conversion paths.

## Trigger / entry point

Crafted metadata causes excessive allocation, repeated cache activity, divide-by-zero faults, or memory growth during parse and conversion.

## Vulnerability mechanism (CWE-style)

The library allocates or computes without effective limits, leaks memory across repeated operations, or fails on malformed arithmetic edge cases. Relevant CWEs include `CWE-400`, `CWE-770`, `CWE-401`, and `CWE-369`.

## Expected impact

Process crash, worker starvation, node instability, and reduced availability for shared ingestion services or batch jobs.

## Risk assessment

Risk scoring used: **Severity (1-5) x Likelihood (1-5) = Risk (1-25)**

Risk bands: **1-5 Low, 6-10 Moderate, 11-15 High, 16-20 Very High, 21-25 Critical**

### Severity

**4 - Major.** The main outcome is denial of service rather than code execution, but service interruption can still be expensive in HPC and cloud ingestion contexts.

### Likelihood

**4 - Likely.** Availability-focused attacks are generally easier to trigger than reliable memory-corruption exploitation, and the CVE list shows several recurring resource-handling weaknesses.

**Risk:** **16 (Very High)**

## Exploitability notes

This threat rises in priority for multi-tenant converters, notebook services, and queue-based ingestion systems where a single malformed file can tie up scarce workers or exhaust per-node memory.

## Detection

Resource-usage telemetry, leak checks in CI, fuzzing with allocation budgets, and tests that enforce strict CPU, memory, and file-size ceilings on malformed inputs.

## Mitigations

Add explicit parser budgets, cap derived allocations and object counts, fail fast on impossible arithmetic, and run untrusted-file processing inside cgroups or equivalent resource-isolated execution.

## Tests / evidence

- Regression tests for `H5C__load_entry`, `H5FL__malloc`, and `H5T__complete_copy`.
- Long-run tests that repeatedly open malformed files to detect leaks and cache growth.
- CVE evidence from `CVE-2025-6817`, `CVE-2025-7068`, and `CVE-2021-46244`.
