# ATK-013: Trojanized HDF5 packages or binaries compromise the host system

**CASSE:** Library • Poisoning • System

**HDF5 SSP vulnerability tags:** `SCD|TCD`

## Preconditions

- Users install HDF5 from packages, containers, wheels, modules, or build outputs whose provenance is not strongly verified.
- The environment trusts the installed artifact enough to execute it during import, link, or runtime initialization.

## Trigger / entry point

A compromised or substituted HDF5 artifact is installed and then loaded by an application, tool, build system, or runtime dependency chain.

## Vulnerability mechanism (CWE-style)

The software supply chain allows unsigned, unverified, or tampered artifacts to be consumed as if they were trusted releases. Relevant CWEs include `CWE-494`, `CWE-353`, and `CWE-829`.

## Expected impact

Arbitrary code execution on developer workstations, CI agents, clusters, and production systems, plus long-lived compromise because the malicious artifact may be redistributed through caches and containers.

## Risk assessment

Risk scoring used: **Severity (1-5) x Likelihood (1-5) = Risk (1-25)**

Risk bands: **1-5 Low, 6-10 Moderate, 11-15 High, 16-20 Very High, 21-25 Critical**

### Severity

**5 - Critical.** A poisoned HDF5 distribution bypasses file-level defenses entirely and compromises the host before any normal parsing safeguards matter.

### Likelihood

**3 - Possible.** Supply-chain attacks are less common than malformed-file attacks, but they remain credible for widely distributed scientific software that is mirrored and repackaged across many channels.

**Risk:** **15 (High)**

## Exploitability notes

This threat matters most where teams pull prebuilt artifacts from multiple package ecosystems or reuse opaque containers and CI outputs without provenance or reproducibility checks.

## Detection

Verify signatures and provenance at install time, compare build outputs against reproducible baselines where available, and continuously inventory deployed HDF5 artifacts with SBOM-backed attestation.

## Mitigations

Sign release artifacts, publish provenance and SBOMs, pin dependencies, require trusted mirrors or registries, and restrict execution privileges for newly introduced HDF5 packages until integrity checks pass.

## Tests / evidence

- Release-pipeline tests that fail unsigned or unverifiable artifacts.
- Reproducibility checks comparing independent builds.
- Deployment audits that confirm installed HDF5 artifacts match approved hashes and provenance attestations.
