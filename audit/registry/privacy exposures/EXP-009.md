# EXP-009: External links reveal filenames and object paths

**HDF5 SSP vulnerability tags:** `FMT|PRV`

**Exposure family:** P2 Structural or on-disk leakage

**Lifecycle stage:** Sharing and publication

**Sensitive asset:** Usernames, project names, mount points, study labels, and object paths encoded in external-link targets.

## Trigger

An HDF5 file containing absolute paths or descriptive relative paths is shared outside the environment where the links were created.

## Exposure surface

The link target strings themselves, plus any logs, validation output, or UI previews that display the filenames and internal object paths.

## Disclosure

Recipients learn internal directory structures, account names, project organization, and HDF5 object naming conventions even if the link target is never accessible.

## Unintended audience

External recipients, archive users, support staff handling broken-link reports, and tools that catalog or preview link metadata.

## Loss / impact

Path disclosure exposes internal naming conventions and identifiers that may aid re-identification, infrastructure mapping, or targeted follow-on attacks.

## Risk assessment

Risk scoring used: **Severity (1-5) x Likelihood (1-5) = Risk (1-25)**
Risk bands: **1-5 Low, 6-10 Moderate, 11-15 High, 16-20 Very High, 21-25 Critical**

### Severity

**3 - Moderate.** The main loss is contextual and operational leakage, but filenames and object paths often encode identities or sensitive study details.

### Likelihood

**2 - Unlikely.** The issue depends on external-link use and subsequent sharing, though descriptive paths are common once links exist.

**Risk:** **6 (Moderate)**

## Detection

Scan for external-link targets that include absolute paths, user home directories, mount names, or descriptive object-path segments. Review exported logs and previews for the same strings.

## Controls/mitigations

### Minimization

Do not embed user-specific or environment-specific paths in files intended for reuse. Prefer neutral packaging or remove external links entirely before export.

### Redaction

Rewrite targets to neutral placeholders or strip them from shared derivatives. Redact path strings in validation logs and user-facing errors.

### Retention / packaging

Keep environment-specific linked files inside private bundles rather than public artifacts. Prevent CI publish steps from releasing files with unresolved descriptive paths.

### Review gate

Publication review shall reject any artifact whose external-link targets disclose usernames, internal mounts, project identifiers, or sensitive object-path semantics.

## Residual exposure / assumptions

Even if the linked file is private or missing, the path string alone can still be sensitive. This entry does not assume actual target compromise, only disclosure of the reference.

## Tests / evidence

- Metadata scan: Detect absolute paths, home-directory segments, and sensitive object-path patterns in external links.
- Artifact inventory: Review exports, previews, and logs that may repeat the same target strings.
- Checklist or release note: Record whether external links were stripped or normalized before sharing.
